Detailed Description

template<typename TRequest, typename TResponse, typename TThreadPool>
struct bond::ext::gRPC::detail::service_unary_call_data< TRequest, TResponse, TThreadPool >

Implementation class that hold the state associated with receiving incomming calls for one method.

There only needs to be one of these per method in a service, and it can be re-used for receiving subsequent calls. A new detail::unary_call_impl is created for each individual call to hold the call-specific data. Once the invocation of the user callback along with the call-specific data has been enqueued in the thread pool, detail::service_unary_call_data re-enqueues itself to get the next call.
